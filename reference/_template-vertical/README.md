# Sibling-vertical scaffold

This folder is **structural scaffolding**, not a working specialist. It exists so that operator (or a future specialist-builder dispatch) can spin up a sibling vertical — vending, mobile-home parks, car washes, route businesses, etc. — using this laundromat-deal-screener as the template.

The discipline transfers; the domain content swaps.

---

## What transfers across verticals

| Layer | Transfers | Adapts |
| --- | --- | --- |
| `identity.md` | Verdict trichotomy, refusal-gate framing, buyer-sub-segment routing pattern, "no insider expertise" stance | Buyer sub-segments (different verticals have different buyer cohorts); job-list (same 5 jobs typical); voice |
| `rules.md` | Always / Never structure, 6 refusal gates verbatim, DON'T-DECIDE-YET column primitive, routing table format | Sub-vertical mapping (each vertical has its own); financing path differences |
| `examples.md` | 2-example minimum (1 PURSUE-with-flags + 1 PASS); fictional-but-realistic discipline | Domain content fully swaps |
| `memo-mode.md` | Memo trigger + 8-section memo structure | Domain content swaps |
| `reference/<vertical>/screening-criteria.md` | 6 dimensions, banded-not-scored, structural-flag-driven verdict | Each dimension's bands are vertical-specific |
| `reference/<vertical>/industry-benchmarks.md` | Comp-band format, source-set provenance, 90-day refresh discipline | Benchmark numbers fully swap |
| `reference/<vertical>/red-flags-checklist.md` | Per-flag template (Look for / Bands / Why it matters / Diligence to retire / DON'T-DECIDE-YET) | Flag set is vertical-specific |
| `reference/<vertical>/valuation-methodology.md` | 5-step method, refuse-without-comps discipline, buyer-sub-segment differences | Multiples + adjustments are vertical-specific |
| `reference/<vertical>/due-diligence-framework.md` | 3-phase structure (Pre-LOI / Post-LOI pre-close / Close + transition), professional-role-per-item | Specific items vertical-specific |
| `reference/<vertical>/sba-financing.md` | SBA 7(a) baseline + buyer-sub-segment routing + lease-term-rule + revenue-verification | Some verticals have additional eligibility constraints (e.g., MHP has tenant-park regulatory considerations) |

## What does NOT transfer

- Specific dollar bands, multiples, or rates from `industry-benchmarks.md` — entirely vertical-specific
- Specific equipment-lifecycle numbers — vertical-specific
- Sub-vertical axes — different verticals have different axes (vending = machine-mix × location-type × route-density; MHP = lot-count × utility-master-vs-direct × tenant-vs-park-owned-homes; car wash = self-service-vs-tunnel-vs-IBA × labor-model × real-estate-posture)
- Regulatory environment — vertical-specific (MHP has state-tenant-park regulations; car wash has water-discharge permits; vending has machine-bonding requirements)
- Buyer cohort variation — different verticals attract different buyer types (MHP attracts more sophisticated buyers; vending attracts route-buyers; car wash attracts capital-heavier buyers)

## Worker dispatch for a sibling vertical

If operator wants a sibling vertical built, the brief should follow the same three-precondition discipline as the laundromat brief:

1. **Buyer narrowing** — which sub-segments of the vertical's buyer cohort
2. **Reference-source path** — worker authors from public sources, or research-claude scout first
3. **Sub-vertical coverage** — which combinations of the vertical's axes

Plus the visual-identity question (per `[[feedback-visual-identity-per-build]]` memory) and the standard refusal-gate set.

The brief should explicitly cite this laundromat-deal-screener as the structural analog, so the worker knows what transfers vs. adapts.

---

## What this scaffold does NOT do

This folder does not produce a working specialist on its own. It exists to:

- Document the structural transfer pattern
- Save operator-time on the next vertical (no need to re-derive the file layout)
- Make explicit which dimensions are vertical-invariant vs. vertical-specific

Building the sibling vertical is a separate brief.
